🚀 Hermes Agent 使用教程

版本 2.0.0 | 开源 AI 代理框架 by Nous Research

简介

Hermes Agent 是一个开源的 AI 代理框架，由 Nous Research 开发。它可以在终端、消息平台和 IDE 中运行，属于与 Claude Code (Anthropic)、Codex (OpenAI) 同类的自主编码和任务执行代理。

Hermes 支持任何 LLM 提供商（OpenRouter、Anthropic、OpenAI、DeepSeek、本地模型等 20+ 提供商），可在 Linux、macOS 和 WSL 上运行。

核心优势

🧠 自我改进

通过技能系统从经验中学习，保存可重用的流程到未来会话

💾 持久记忆

跨会话记忆用户信息、偏好、环境细节和学到的经验

🌐 多平台网关

同一代理运行在 Telegram、Discord、Slack、WhatsApp 等 10+ 平台

🔧 提供商无关

在工作流中随时切换模型和提供商，自动轮换多个 API 密钥

👤 多配置文件

运行多个独立的 Hermes 实例，每个都有独立的配置、会话和技能

🔌 可扩展

支持插件、MCP 服务器、自定义工具、Webhook 触发器和完整 Python 生态

快速开始

安装

# Linux/macOS
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash

# Windows (PowerShell)
irm https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.ps1 | iex

基本使用

# 交互式聊天（默认）
hermes

# 单次查询
hermes chat -q "法国的首都是哪里？"

# 设置向导
hermes setup

# 切换模型/提供商
hermes model

# 健康检查
hermes doctor

CLI 命令参考

全局标志

hermes [flags] [command]
  --version, -V            显示版本
  --resume, -r SESSION     恢复指定 ID 或标题的会话
  --continue, -c [NAME]    按名称恢复，或恢复最近的会话
  --worktree, -w           隔离的 git worktree 模式（并行代理）
  --skills, -s SKILL       预加载技能（逗号分隔或重复）
  --profile, -p NAME       使用命名配置文件
  --yolo                   跳过危险命令确认

聊天命令

hermes chat [flags]
  -q, --query TEXT          单次查询，非交互式
  -m, --model MODEL        模型（如 anthropic/claude-sonnet-4）
  -t, --toolsets LIST      逗号分隔的工具集
  --provider PROVIDER      强制使用提供商
  -v, --verbose            详细输出

配置命令

hermes setup [section]      交互式向导
hermes model                交互式模型/提供商选择器
hermes config               查看当前配置
hermes config edit         在 $EDITOR 中打开 config.yaml
hermes config set KEY VAL  设置配置值
hermes config check        检查缺失/过时的配置
hermes doctor [--fix]      检查依赖和配置

配置说明

关键路径（Windows）

C:\Users\<用户名>\.hermes\config.yaml       主配置文件
C:\Users\<用户名>\.hermes\.env              API 密钥和秘密
C:\Users\<用户名>\.hermes\persona.md        代理个性（可选）
%USERPROFILE%\.hermes\skills\             已安装的技能
%USERPROFILE%\.hermes\sessions\           会话记录
%USERPROFILE%\.hermes\logs\               网关和错误日志

定时任务（Cron）

使用 cronjob 工具管理定时任务。

调度格式

格式说明
30m每 30 分钟
every 2h每 2 小时
0 9 * * *每天 9:00

Cron 命令

hermes cron list            列出任务
hermes cron create SCHED    创建任务
hermes cron edit ID         编辑任务
hermes cron pause/resume ID 控制状态
hermes cron remove ID       删除任务

语音功能

语音转文本（STT）

提供商优先级：

  • 本地 faster-whisper - 免费：pip install faster-whisper
  • Groq Whisper - 免费层级：设置 GROQ_API_KEY
  • OpenAI Whisper - 付费：设置 VOICE_TOOLS_OPENAI_KEY

文本转语音（TTS）

提供商免费？
Edge TTS✅ 是（默认）
ElevenLabs免费层级
NeuTTS（本地）✅ 是

语音命令

/voice on      # 语音到语音模式
/voice tts      # 始终使用语音回复
/voice off      # 关闭语音

故障排除

常见问题

  • 工具不可用：运行 hermes tools 检查是否启用
  • 模型问题：运行 hermes doctor 检查配置
  • 更改未生效：退出并重新启动 Hermes
  • Windows HTTP 400：确保 config.yaml 以 UTF-8 无 BOM 保存

网关日志

grep -i "failed to send\|error" ~/.hermes/logs/gateway.log | tail -20
