02
SETUP

安装与配置

Claude Code 是 Anthropic 推出的官方命令行工具，支持 macOS、Windows 和 Linux。

terminal 
# macOS / Linux
curl https://claude.ai/install.sh | sh

# Windows PowerShell
irm https://claude.ai/install.ps1 | iex

# npm 全局安装
npm install -g @anthropic-ai/claude-code

# 验证
claude --version

初始化流程

设置 API Key

首次运行提示输入 Anthropic API Key，或预设环境变量 ANTHROPIC_API_KEY

选择权限模式

accept-all 自动批准 / plan 先规划 / default 逐项确认

初始化项目

运行 /init 创建 CLAUDE.md 让 Claude 了解项目结构

TIP

使用 claude config 进入交互式配置界面，可视化设置所有选项。

配置文件位置

类型路径作用域
全局设置~/.claude/settings.json所有项目
本地设置.claude/settings.local.json当前项目
项目说明CLAUDE.md当前项目
MCP 配置~/.claude/.mcp.json全局 MCP

EXTEND

MCP 服务器

MCP (Model Context Protocol) 是 Anthropic 推出的扩展协议，连接外部工具和数据源。

~/.claude/.mcp.json 
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-filesystem", "/path"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-github"],
      "env": { "GITHUB_TOKEN": "xxx" }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-server-postgres"]
    }
  }
}
filesystem

文件系统访问，配置允许目录

github

GitHub API，仓库/PR/Issues

postgres

PostgreSQL 数据库查询

browser

浏览器自动化，网页抓取

安全

MCP 服务器可访问外部资源，只添加可信来源。

06
AUTOMATE

Hooks 配置

Hooks 是事件钩子系统，在特定事件发生时自动执行脚本。

事件触发时机用途
PreToolUse工具执行前拦截、修改参数
PostToolUse工具执行后处理结果、触发动作
Notification通知事件自定义通知方式
Stop会话结束清理、保存状态
PrePrompt用户提交前修改 prompt
settings.json 
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": [{
        "type": "command",
        "command": "git add $TOOL_INPUT_FILE_PATH"
      }]
    }]
  }
}

环境变量

  • $TOOL_NAME — 工具名称
  • $TOOL_INPUT — 输入参数 JSON
  • $TOOL_OUTPUT — 输出结果
  • $TOOL_INPUT_FILE_PATH — 文件路径
  • $SESSION_ID — 会话 ID

09
MASTER

最佳实践

CLAUDE.md 配置

CLAUDE.md 
# Project: My App

## 项目概述
Next.js 14 + TypeScript + Tailwind 电商前端

## 代码规范
- 函数组件 + hooks
- 文件命名: kebab-case
- 组件命名: PascalCase

## 重要文件
- 入口: src/app/layout.tsx
- API: src/app/api/

## 禁止事项
- 不修改 src/lib/auth.ts
- 不删除 __tests__/ 目录

## 工作流程
- 提交前运行 npm run lint && npm test
- PR 需关联 Linear issue
分步指令

复杂任务拆分，便于中途检查调整

精确搜索

用 Glob/Grep 指定路径，提高准确度

上下文压缩

定期 /compact，避免 token 超限

权限预配置

预设常用工具权限，减少中断

安全提醒

不要让 Claude：修改生产配置、删除数据库、推送未测试代码、暴露 API Key。

终极技巧

让 Claude Code 做它擅长的事——理解代码、生成代码、调试问题。决策和判断交给人类。
