第一部分  Git 命令指南

Git 是目前世界上最先进的分布式版本控制系统。本章从基础到高级，系统化梳理 Git 的常用命令与工作流程。

1.1  基础配置与初始化

安装 Git 后的第一步是配置用户信息，这是每次提交记录的标识：

# 配置全局用户名和邮箱 git config --global user.name "Your Name" git config --global user.email "email@example.com" # 查看配置 git config --list git config user.name # 设置默认分支名（Git 2.28+） git config --global init.defaultBranch main
命令说明
git init在当前目录初始化一个新的 Git 仓库
git clone <url>克隆远程仓库到本地
git clone <url> <dir>克隆到指定目录
git status查看工作区状态
git status -s精简模式查看状态

1.2  日常基本操作

Git 的日常操作围绕「工作区 → 暂存区 → 仓库」三区模型展开：

# 添加文件到暂存区 git add <file> # 添加单个文件 git add . # 添加所有变更 git add -p # 交互式分块暂存 # 提交到仓库 git commit -m "message" # 直接提交 git commit -am "message" # 跳过 add（仅跟踪过的文件） git commit --amend # 修改上一次提交 git commit --amend -m "新消息" # 修改上一次提交的消息
命令说明
git log查看提交历史
git log --oneline单行显示提交历史
git log --graph --oneline --all图形化显示所有分支历史
git diff查看工作区与暂存区的差异
git diff --staged查看暂存区与最近提交的差异
git show <commit>查看某次提交的详情
git blame <file>查看文件每行最后的修改者

1.3  分支管理

分支是 Git 的核心能力，支持并行开发与特性隔离：

# 创建与切换 git branch <name> # 创建分支 git switch <name> # 切换分支（推荐） git checkout <name> # 切换分支（传统方式） git switch -c <name> # 创建并切换 git checkout -b <name> # 创建并切换（传统方式） # 查看分支 git branch # 本地分支列表 git branch -r # 远程分支列表 git branch -a # 所有分支 git branch -v # 显示最近提交 # 删除与重命名 git branch -d <name> # 删除已合并的分支 git branch -D <name> # 强制删除 git branch -m <old> <new> # 重命名分支
提示：Git 2.23+ 推荐使用 git switch 和 git restore 替代 git checkout 的部分功能，语义更清晰。

1.4  合并与变基

合并（merge）和变基（rebase）是整合分支的两种方式：

# 合并 git switch main # 先切到目标分支 git merge <branch> # 将 branch 合并到当前分支 git merge --no-ff <branch> # 禁用快进合并，保留分支历史 # 变基 git switch feature git rebase main # 将 feature 变基到 main 之上 # 交互式变基 git rebase -i HEAD~3 # 修改最近 3 次提交 git rebase -i <commit> # 从某次提交开始变基
命令说明
git merge --abort中止冲突中的合并
git rebase --abort中止冲突中的变基
git rebase --continue解决冲突后继续变基
git rebase --skip跳过当前提交
提示：公共分支（如 main）上不要 rebase。变基会重写提交历史，仅用于本地或私有特性分支。

1.5  远程仓库操作

# 远程仓库管理 git remote -v # 查看远程仓库 git remote add origin <url> # 添加远程仓库 git remote remove <name> # 删除远程仓库 git remote set-url <name> <url> # 修改远程地址 # 推送与拉取 git push origin main # 推送到远程 git push -u origin main # 推送并设置上游 git push origin --delete <branch> # 删除远程分支 git push --tags # 推送标签 git pull # 拉取并合并（fetch + merge） git pull --rebase # 拉取并变基（推荐） git fetch # 仅获取远程更新，不合并 git fetch --prune # 同步删除已不存在的远程分支

1.6  撤销与恢复

Git 提供了多层级的撤销能力，覆盖从工作区到已推送的各个阶段：

# 工作区撤销 git restore <file> # 丢弃工作区的修改 git checkout -- <file> # 传统方式 # 暂存区撤销 git restore --staged <file> # 取消暂存 git reset HEAD <file> # 传统方式 # 提交撤销 git reset --soft HEAD~1 # 撤销提交，保留修改到暂存区 git reset --mixed HEAD~1 # 撤销提交，保留修改到工作区（默认） git reset --hard HEAD~1 # 彻底丢弃提交和修改（⚠️ 危险） # 已推送的提交 git revert HEAD # 创建一个新的撤销提交（安全） git revert <commit> # 撤销指定提交

1.7  暂存与标签

命令说明
git stash暂存当前工作区修改
git stash list查看暂存列表
git stash pop恢复并删除最近的暂存
git stash apply恢复但不删除暂存
git stash drop删除指定的暂存
git tag <name>创建轻量标签
git tag -a <name> -m "msg"创建附注标签
git tag列出所有标签
git tag -d <name>删除本地标签

1.8  高级技巧

Cherry-Pick：选择性移植提交

将某个分支上的特定提交应用到当前分支：

git cherry-pick <commit> # 应用某个提交 git cherry-pick <A>..<B> # 应用 A 到 B 之间的所有提交（不含 A） git cherry-pick --continue # 解决冲突后继续 git cherry-pick --abort # 中止 cherry-pick

Bisect：二分法定位问题

通过二分搜索快速定位引入 Bug 的提交：

git bisect start # 开始二分查找 git bisect bad # 当前版本有问题 git bisect good <commit> # 某个旧版本是好的 # Git 会 checkout 一个中间提交 # 测试后标记： git bisect good # 此提交正常 git bisect bad # 此提交有问题 # 重复直到找到首个坏提交 git bisect reset # 结束 bisect

子模块与工作树

# 子模块 git submodule add <url> <path> # 添加子模块 git submodule update --init # 初始化并更新子模块 git clone --recurse-submodules <url> # 克隆时包含子模块 # 工作树（同时检出多个分支） git worktree add <path> <branch> # 在另一目录检出分支 git worktree list # 列出所有工作树 git worktree remove <path> # 移除工作树

1.9  常见场景速查

命令说明

git log --since="2 weeks ago"查看两周内的提交
git shortlog -sn按提交者统计提交数
git gc清理和优化本地仓库

第二部分  GitHub CLI (gh) 使用指南

GitHub CLI（gh）是 GitHub 官方的命令行工具，让你无需离开终端即可完成 GitHub 上的绝大多数操作。

2.1  安装与认证

# 安装（各平台） # macOS: brew install gh # Windows（WinGet / 直接下载）: winget install --id GitHub.cli # 或从 https://cli.github.com/ 下载安装包 # Linux (Debian/Ubuntu): curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null sudo apt update && sudo apt install gh

安装后的第一步是认证：

# 交互式登录（推荐） gh auth login # 非交互式（Token 方式） gh auth login --with-token < token.txt # 查看认证状态 gh auth status # 切换账号 gh auth switch
提示：gh auth login 支持 GitHub.com 和 GitHub Enterprise Server。通过浏览器 OAuth 流程或 Personal Access Token 完成认证。

2.2  仓库管理

# 克隆仓库 gh repo clone owner/repo # 克隆到当前目录 gh repo clone owner/repo ./dir # 克隆到指定目录 # 创建仓库 gh repo create <name> # 交互式创建 gh repo create <name> --public # 创建公开仓库 gh repo create <name> --private # 创建私有仓库 gh repo create <name> --clone # 创建并克隆到本地 # 查看仓库信息 gh repo view # 查看当前仓库 gh repo view owner/repo # 查看指定仓库 gh repo view --web # 在浏览器中打开

Fork 与同步

# Fork 仓库 gh repo fork # Fork 当前仓库 gh repo fork owner/repo # Fork 指定仓库 gh repo fork --clone # Fork 后自动克隆到本地 # 同步上游（需先配置上游 remote） gh repo sync # 同步当前分支 gh repo sync main # 同步指定分支 gh repo sync -b main --source owner/repo # 从指定仓库同步
命令说明
gh repo list列出你的仓库
gh repo list owner列出某用户的仓库
gh repo rename <name>重命名仓库
gh repo delete删除仓库（⚠️ 不可恢复）
gh repo archive归档仓库
gh repo edit编辑仓库设置（描述、主题等）

2.3  Issue 管理

# 创建 Issue gh issue create # 交互式创建 gh issue create --title "标题" --body "内容" gh issue create --label "bug" --assignee @me # 查看 Issue gh issue list # 列出开放的 Issue gh issue list --state closed # 列出已关闭的 gh issue list --label "bug" # 按标签筛选 gh issue list --assignee @me # 指派给我的 gh issue list --limit 30 # 限制数量 gh issue view <number> # 查看 Issue 详情 gh issue view <number> --comments # 包含评论 gh issue view <number> --web # 在浏览器打开
命令说明
gh issue close <number>关闭 Issue
gh issue reopen <number>重新打开 Issue
gh issue comment <number> -b "msg"添加评论
gh issue edit <number> --add-label "feature"添加标签
gh issue edit <number> --remove-label "bug"移除标签
gh issue pin <number>置顶 Issue
gh issue unpin <number>取消置顶
gh issue lock <number>锁定 Issue（限制评论）

2.4  Pull Request

Pull Request（PR）是 GitHub 协作的核心功能，gh 提供了完整的 PR 工作流支持：

# 创建 PR gh pr create # 交互式创建 gh pr create --title "标题" --body "描述" gh pr create --draft # 创建 Draft PR gh pr create --base main --head feature # 指定基准和比较分支 gh pr create --label "review" --assignee @me --reviewer @user # 查看 PR gh pr list # 列出开放的 PR gh pr list --state merged # 已合并 gh pr list --author @me # 我创建的 gh pr list --review-requested @me # 需要我审查的 gh pr list --search "keyword" # 搜索 gh pr view <number> # 查看详情 gh pr view <number> --comments # 含评论 gh pr view <number> --web # 浏览器打开
命令说明
gh pr checkout <number>将 PR 检出到本地
gh pr diff查看 PR 的 diff
gh pr review --approve批准 PR
gh pr review --comment -b "msg"评论审查意见
gh pr review --request-changes请求更改
gh pr merge合并 PR（交互式）
gh pr merge --squashSquash 合并
gh pr merge --rebaseRebase 合并
gh pr close关闭 PR（不合并）
gh pr ready将 Draft PR 标记为 Ready
提示：gh pr merge 默认使用 merge commit。通过 --squash 或 --rebase 可切换合并策略。合并前建议先通过 CI 检查。

2.5  代码审查与讨论

# PR 审查 gh pr review --approve # 批准 gh pr review --comment -b "LGTM!" # 评论 gh pr review --request-changes -b "请修改" # 请求更改 # 评论与讨论 gh pr comment <number> -b "评论内容" # 添加 PR 评论 gh issue comment <number> -b "评论内容" # 添加 Issue 评论 # 查看检查与状态 gh pr checks <number> # 查看 CI 检查状态 gh pr status # 查看当前分支 PR 状态

2.6  GitHub Actions

通过 gh 可以直接触达 GitHub Actions 的完整能力：

# 查看 Workflow gh workflow list # 列出仓库中的 workflows gh workflow view <id> # 查看 workflow 详情 # 触发 Workflow gh workflow run <id> # 手动触发 workflow gh workflow run <id> --ref main # 指定分支 gh workflow run <id> -f key=value # 传入参数 # 查看运行记录 gh run list # 最近的运行记录 gh run view <id> # 查看运行详情 gh run view <id> --log # 查看日志 gh run view <id> --log-failed # 仅查看失败的步骤日志 # 其他操作 gh run download <id> # 下载 artifacts gh run rerun <id> # 重新运行 gh run cancel <id> # 取消运行 gh run watch <id> # 实时查看运行状态

2.7  Gist 管理

Gist 是 GitHub 提供的轻量级代码片段管理服务：

# 创建 Gist gh gist create <file> # 创建公开 Gist gh gist create <file> --public # 明确指定公开 gh gist create <file1> <file2> # 多文件 Gist gh gist create -d "描述" <file> # 带描述 # 查看与管理 gh gist list # 列出你的 Gist gh gist list --public # 仅公开的 gh gist list --secret # 仅私有的 gh gist view <id> # 查看 Gist 内容 gh gist edit <id> <file> # 编辑 Gist gh gist delete <id> # 删除 Gist

2.8  配置与别名

# 配置 gh config set editor vim # 设置默认编辑器 gh config set git_protocol ssh # 使用 SSH 协议 gh config set browser chrome # 设置默认浏览器 # 别名（简化命令） gh alias set prc "pr create" # gh prc = gh pr create gh alias set co "pr checkout" # gh co = gh pr checkout gh alias set il "issue list" # gh il = gh issue list gh alias set pc "pr create --fill" # 自动填充标题和 body # 查看别名 gh alias list # 列出所有别名

2.9  扩展机制

GitHub CLI 支持通过扩展（extensions）增强功能。扩展可以是任意可执行程序：

# 安装扩展 gh extension install <owner>/<repo> # 从 GitHub 安装 # 常用扩展 gh extension install dlvhdr/gh-dash # 增强的仪表盘 gh extension install seachicken/gh-poi # 清理已合并的本地分支 gh extension install meiji163/gh-notify # 通知管理 # 管理扩展 gh extension list # 列出已安装的扩展 gh extension upgrade <name> # 升级扩展 gh extension remove <name> # 卸载扩展

2.10  综合工作流示例

以下展示一个完整的日常协作流程：

# 1. 克隆仓库并创建特性分支 gh repo clone owner/project cd project git switch -c feat/new-feature # 2. 开发并提交 echo "code" >> file.js git add . && git commit -m "feat: add new feature" # 3. 推送并创建 PR git push -u origin feat/new-feature gh pr create --title "新功能" --body "实现描述..." --label "enhancement" # 4. 提交审查后合并 gh pr review --approve gh pr merge --squash # 5. 清理本地分支 git switch main git pull --rebase git branch -d feat/new-feature
